# goals — プロジェクトの目的と完成形

作成日時: 2026-07-04 22:01
更新日時: 2026-07-04 22:01

## 目的

興味分野(半導体株、GPU、メモリ、AI、ローカルLLM 等)のニュースを、ローカルLLM + Web検索MCP で**自動監視・蓄積・深堀り**できるダッシュボード型ニュースアプリを作る。

- 複数カテゴリを数分おきに自動チェックし、タイトルを即時にダッシュボードへ流す。
- クリックで初めて LLM 要約を生成する on-demand 加工(全記事を LLM に通さない)。
- 取得したニュースは Markdown として蓄積し、RAG と深堀りチャットの材料にする。
- カテゴリごとに「今の状況」ロールアップを LLM が自動更新する。

## 完成形のイメージ

- kanban 風のカテゴリ列 UI に新着タイトルがリアルタイム(SSE)で流れ込む。
- カードクリックで要約・要点・impact・タグが右パネルに表示される。
- カードの保存(ピン留め)/非表示(tombstone で復活しない削除)ができる。
- 「このニュースを深堀り」から 35B モデルのエージェンティックチャット(vault_search + Web検索)が起動し、引用付きで回答する。
- トレイ常駐で裏で動き続け、config(categories.yaml)の編集だけでカテゴリを追加できる。

## 重視する価値(設計の核)

1. **MD が真実の源、SQLite は再構築可能な派生索引。** 索引が壊れても MD から全再構築できる。
2. **二層カデンス。** 安いタイトル取り込み(高頻度・LLMなし)と高い LLM 加工(オンデマンド)を分離。
3. **ソフト削除 + tombstone。** 削除した記事は再取得で復活しない。
4. **既存資産の再利用。** rss-digest(スケジューラ・トレイ常駐)と mem-chat(検索・保存・埋め込み・SSE)を土台にし、新規実装は config 層とダッシュボード UI に絞る。

## 制約・前提

- 実行環境: ローカル(RTX PRO 5000 / 48GB VRAM)。9B + 35B の2モデル常駐で VRAM 予算 ~31GB + KV キャッシュ。
- LLM は OpenAI 互換エンドポイント(llama.cpp llama-server)経由。モデルは `models/` 配下の GGUF。
- 詳細は [../news-picker-spec.md](../news-picker-spec.md) を参照。
