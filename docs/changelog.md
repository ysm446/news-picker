## 未リリース

- **環境設定**: 設定モーダルをタブ化(カテゴリ / 環境設定)。設定は `data/settings.json` に保存(`GET/PUT /settings`)。
  - **ニュース見出しの日本語訳**: オンにすると 9B が英語等の見出しを日本語化(キュレーション採点と同じ呼び出しに相乗り。新着から自動適用、既存分は `POST /admin/curate-now?category=&force=true` で再採点)。カードは訳を表示し、ツールチップで原文確認可。
  - **ノイズ閾値**(既定30)と**記事の保持日数**(既定14日)を設定画面から変更可能に。

- カテゴリ設定の実体を `data/categories.yaml` に移動(個人データのため git 管理外に)。リポジトリには雛形 `config/categories.example.yaml` のみ。初回起動時に自動コピー。
- 詳細パネルの「閉じる」ボタンを右端に移動。

- **35B のオンデマンド化**: 起動時は 9B のみロードし VRAM 消費を約31GB→約9GBに削減。ステータスバーの「35B」ランプがトグルボタンになり、クリックでロード/アンロード(ロードは数十秒、切り替え中は橙点滅)。35B オフの間は深堀りチャットを 9B が代行し、チャット画面に使用モデルを表示。

- **キュレーション(自動ノイズ除去)**: 取り込んだ記事を 9B がカテゴリ関連度 0〜100 で一括採点(タイトル配信はブロックしない後追い方式)。関連度30未満は「ノイズを隠す」フィルタ(既定ON)で非表示。削除ではないのでチェックを外せばいつでも見える。保存済み記事は対象外。カードに採点値を表示。既存記事の一括採点は `POST /admin/curate-now?category=`。
- **カテゴリ管理 UI**: トップバーの「設定」からカテゴリの追加・編集・削除が可能に(検索クエリ、取得間隔、impact 軸、要約プロンプト等)。変更は categories.yaml に書き戻され、ワーカーが即再構築される。削除しても記事データは残り、再作成すれば再表示される。
- 下部ステータスバーを追加: CPU / RAM / GPU / VRAM をメーターバー付きで右端に表示、左端に 9B・35B サーバーの稼働ランプ(2秒間隔)。トップバーの「リソース表示/非表示」で切り替え(設定は記憶される)。lm-chat の /system/resources を移植。
- 起動直後の空白時間を改善: バックエンド自動起動が終わるまで UI が2秒間隔で再試行し「接続しています...」を表示。SSE 再接続時にもデータを取り直す。ウィンドウは描画準備完了後に表示(白画面のちらつき防止)。
- ウィンドウの既定サイズを 1920x1080 に変更(画面に収まらない場合は自動縮小)。
- 修正: カテゴリ設定の「検索クエリ」欄で改行やスペースが入力できなかった問題(入力のたびに正規化していたのを保存時のみに変更。impact 欄のカンマも同様)。
- 修正: Electron (file://) 起動で画面が真っ白になる問題(Vite の base を相対パスに変更)。
- カテゴリ要約に Markdown 記法が混ざらないようプロンプトを調整。

## [0.1.0] - 2026-07-05 00:06

初版。仕様書(docs/news-picker-spec.md)の実装フェーズ1〜7を完了。

### 機能

- **ダッシュボード**: カテゴリ列(kanban 風)に新着ニュースのタイトルがリアルタイム(SSE)で流れ込む。フィルタ(期間 / impact / ティッカー・企業名・タグ / 保存のみ)。
- **取り込み**: カテゴリごとのポーリング(ddgs ニュース検索、API キー不要)。URL 正規化 + dedup。config/categories.yaml でカテゴリ追加可能(設定再読込ボタン / POST /admin/reload-config)。
- **詳細生成**: カードクリックで 9B が要約・要点・エンティティ・impact・タグを生成(thinking 無効 + 構造化出力)。記事は MD として news-vault に蓄積され、FTS + 768次元埋め込みで索引される。
- **保存 / 非表示**: 非表示・パージ済み記事は tombstone により再取得でも復活しない。自動整理(既定14日、saved は対象外)。
- **カテゴリ要約**: 新着トリガ + デバウンスで「今の状況」ロールアップを自動更新、列ヘッダに表示。
- **深堀りチャット**: 35B が vault_search(ハイブリッド検索 + 時間減衰)と web_search を自律的に使い、出典 URL 付きで回答。記事からもトップバーからも起動可能。
- **運用**: start.bat / stop.bat。Electron 単体起動でもバックエンドと llama-server を自動起動。ウィンドウを閉じてもトレイ常駐で取り込み継続。SQLite 索引は MD から全再構築可能(POST /admin/rebuild-index)。
