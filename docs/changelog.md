## 未リリース

- **修正: 詳細生成の「peg-native format」500 エラーの成功率改善**: リトライを1回→2回に増やし、リトライ時は temperature を 0.2 に下げ (出力の安定化)、max_tokens を 4096 に拡大 (JSON の途中切断対策) して引き直すように。あわせて詳細パネルの生成失敗表示に「再試行」ボタンを追加 (従来は開き直すしか再試行手段がなかった)。

- **コードレビューに基づく修正一式** (2026-07-12):
  - **「MD が真実の源」の整合性を回復**: 保存・👍の状態変更を記事 MD へ書き戻し、非表示/興味なしで MD を削除するように(従来は DB のみ更新で、rebuild-index 後に「保存が巻き戻って自動整理に消される」「非表示記事が復活する」問題があった)
  - **保存時に未 enrich なら自動で詳細生成**: 保存 = vault に MD として確実に書き残すセマンティクスに(plan.md の予定項目)
  - **rebuild-index の強化**: tombstone 済み記事の MD を復元対象から除外、全体を1トランザクション化(途中失敗で半壊 DB にならない・稼働中ワーカーとの ID 衝突防止)、埋め込み (vec_articles) も復元するように(従来は rebuild で意味検索が黙って壊れていた)
  - **enrich の部分失敗を自己修復可能に**: 埋め込み計算 → MD 書き出し → DB 確定の順に変更(従来は DB 確定が先で、後段失敗時に「enrich 済みだが MD/埋め込みなし」で恒久確定していた)
  - **セキュリティ**: 記事リンクを OS で開く際に http(s) 以外のプロトコルを拒否(細工されたフィード経由の任意プロトコルハンドラ起動を防止)。サムネイル取得も http(s) 限定に (file:// / UNC 拒否)。ウィンドウ自体の外部サイト遷移も禁止
  - **深堀りチャットの安定化**: 生成中にバックエンドが落ちても送信ボタンが固まらないように(busy 解除漏れ)、パネルを閉じたら生成を中断(LLM 占有の解消)、別記事への深堀り切替で会話履歴をリセット(旧記事の文脈混入を防止)
  - **終了処理**: トレイ「終了」でバックエンドが応答しない異常系でも、llama-server がプロセスツリーごと止まり VRAM を掴んだまま残らないように
  - **SSE/enrich キューの遅延解消**: ワーカースレッドからのイベント配信・enrich 投入を call_soon_threadsafe 化(アイドル時にクリックしても enrich 開始が遅れることがあった)

- 詳細パネルの操作アイコン(保存 / 良い記事 / 興味なし)をヘッダーからタイトル直下のメタ行(ソース・時刻の行)右端へ移動。カードと同じ配置になり、画像の下で記事情報とセットで見えるように。ヘッダーは「閉じる」のみに。
- **impact 機能を廃止**: impact は詳細生成(クリック)した記事にしか付かず、未読の大多数には無いためフィルタとして機能せず、読了後のバッジも情報量が薄かった。フィルタバーのドロップダウン、カード・詳細パネルのバッジ、カテゴリ設定の「impact の選択肢」欄、詳細生成での impact 判定を削除(生成がわずかに軽くなる)。DB の impact 列と既存 MD の frontmatter は互換のため残す(rebuild-index に影響なし)。
- **修正: 古い保存済みカードが画面から見えなくなる問題**。各列は「新しい順 60 件」しか取得しないため、保存カードはデータとしては残っていても(自動整理の対象外)、新着に押し出されて表示から漏れていた。「保存のみ」フィルタを ON にすると保存済みをサーバーから全件取得して表示するように修正。
- **カテゴリの表示/非表示**: フィルタバーに「カテゴリ」ボタンを追加。ポップアップのチェックボックスで列ごとに表示を切り替えられる。**非表示中のカテゴリは取り込み・要約も停止**する(categories.yaml の `enabled` に保存)。非表示があるときはボタンがアクセント色。
- フィルタバーの「保存のみ」「ノイズを隠す」ボタンを「フィルタ」ボタン1つに統合。押すとポップアップでチェックボックスが開く。既定と異なる状態(保存のみ ON / ノイズ表示中)のときはボタンがアクセント色になる。
- アプリの多重起動を防止。2回目に起動しようとすると、新しいインスタンスは開かず既存のウィンドウが前面に出る(トレイ格納中でも復帰)。
- **カテゴリ列をドラッグで並び替え**: 列ヘッダー(タイトル行)をつかんで別の列にドロップするとその位置に移動。順序は categories.yaml に保存され(`POST /categories/reorder`)、取り込みワーカーは止めずに表示順だけ入れ替える。
- 詳細パネルにもサムネイル画像を表示(本文の最上部にフル幅)。環境設定の「カードにサムネイル画像を表示する」に連動。
- **カテゴリ列に「今すぐ更新」ボタンを追加**: 列ヘッダーの設定アイコンの隣。ポーリングを待たずに検索 / RSS 取り込み+採点を即時実行する(`POST /admin/ingest-now` を利用)。実行中はアイコンが回転。
- **トレイの「終了」を完全終了に**: `POST /admin/shutdown` を追加し、Electron の終了時にバックエンドへ停止を依頼 → バックエンドが llama-server(誰が起動したものでも両ロール)を止めてから自分も終了する。バックエンドが外部起動でも全部きれいに止まり、llama-server が VRAM を掴んだまま残る問題も解消。stop.bat は強制リセット用として残る。(環境設定に一時追加した「バックエンドを再起動」ボタンは、黒いコンソールが出る問題もあり削除)
- **修正: 詳細生成が「llama-server 500 (peg-native format)」で失敗することがある問題**。新しめの llama.cpp (b98xx~) は生成出力がテンプレートの期待形式にパースできないと 500 を返すようになった(確率的事象)。該当エラー時に1回リトライを追加し、enrich の max_tokens を 1024→2048 に拡大(長い記事で JSON が途中で切れるのを防止)。
- **カードにサムネイル画像**: ニュース元の画像(ddgs 検索の image / RSS の media サムネイル)をカードのタイトル上に表示。画像は初回表示時にバックエンドが取得して長辺 480px の JPEG に縮小し `data/cache/images/` にキャッシュ(ホットリンクしない)。記事のパージ・非表示と連動して削除。小さすぎる画像(ロゴ等)は除外し、取得失敗時は画像なしカードに自動フォールバック。vault MD の frontmatter にも image_url を記録(rebuild 対応)。環境設定に「カードにサムネイル画像を表示する」トグルを追加(既定オン)。※反映は新規取り込み分から
- **RSS/Atom フィード取り込み**: カテゴリ設定に「RSS/Atom フィード (1行1URL)」欄を追加。検索 (ddgs) と併用でき、フィードのみのカテゴリも作れる(ニュース索引に載らないブログ・GitHub Releases 等を拾える)。取得記事は検索と同じ経路(dedup / 関連度採点 / 日本語訳 / SSE)に合流。ETag/Last-Modified の条件付き GET で未更新フィードは読み飛ばす。雛形の local-llm カテゴリに llama.cpp リリースフィードの例を追加。
- **保存をトグル化**: 保存済み記事の保存ボタン(カード / 詳細パネル)をもう一度押すと解除できるように(👍と同じトグル式)。解除すると自動整理の対象に戻る。保存中はボタンが緑の塗りつぶしブックマークになる。
- **深堀りチャット中も記事詳細を並列表示**: 深堀りチャットを開くと、記事詳細パネルがチャットの左に並んで両方見えるように(従来は排他表示で、チャット中に元記事が見えなかった)。詳細は中央モーダル化も試したが、この並列表示のため右の縦長パネルに戻した。
- **詳細パネルの生成待ちを改善**: 要約の生成中も、検索時に取得済みの抜粋(スニペット)を即表示して読み始められるように。要約欄はスケルトン(点滅プレースホルダー)+「生成中...」表示。「深堀り」ボタンは全幅をやめて右寄せに。
- 詳細パネルのヘッダーに評価アイコン(保存 / 良い記事 / 興味なし)を追加。カードと同じくキュレーション学習に反映され、保存/評価の状態はパネルにも即時反映。「興味なし」を押すとパネルは閉じる。

- ステータスバーの「接続中」表示を廃止(正常時は情報量がないため)。SSE が切断された時のみ「再接続中...」を表示。
- **F12 でスクリーンショット**: コンテンツ部分を PNG で `data/screenshots/` に保存(ファイル名は日時)。あわせてウィンドウの既定サイズをコンテンツ基準の 1920x1080 に変更(従来は枠込みだったため撮影画像が僅かに小さかった)。
- **設定モーダルを2ペイン化**: タブをやめ、左にセクションナビ(カテゴリ / 環境設定)、右にコンテンツという構成に変更。モーダルを 760px に拡大し高さを固定、右ペインのヘッダーにセクション名と×ボタンを表示。カテゴリ編集フォームも右ペイン内で開く。
- **カード UI の全面調整**: ホバー時の操作アイコン(保存/👍/興味なし/非表示)がタイトルに重なる問題を修正 — オーバーレイをやめてメタ行(ソース・時刻の行)右端の常設スロットに移動し、ホバー/キーボードフォーカスで表示。あわせて詰まり感を解消: カード内外の余白を一段階拡大、カード・タグ・関連度・impact の枠線を廃止して背景色チップに変更、未読表現を左ボーダーに一本化(ドット廃止)、「保存済み」をアイコン化、タグ表示は3個まで、カテゴリ要約の折りたたみを2行に、タイトルを 13.5px に拡大。

## [0.1.0] - 2026-07-05 00:06

初版。仕様書(docs/news-picker-spec.md)の実装フェーズ1〜7を完了。

### 機能

- **ダッシュボード**: カテゴリ列(kanban 風)に新着ニュースのタイトルがリアルタイム(SSE)で流れ込む。フィルタ(期間 / impact / ティッカー・企業名・タグ / 保存のみ)。
- **取り込み**: カテゴリごとのポーリング(ddgs ニュース検索、API キー不要)。URL 正規化 + dedup。config/categories.yaml でカテゴリ追加可能(設定再読込ボタン / POST /admin/reload-config)。
- **詳細生成**: カードクリックで 9B が要約・要点・エンティティ・impact・タグを生成(thinking 無効 + 構造化出力)。記事は MD として news-vault に蓄積され、FTS + 768次元埋め込みで索引される。
- **保存 / 非表示**: 非表示・パージ済み記事は tombstone により再取得でも復活しない。自動整理(既定14日、saved は対象外)。
- **カテゴリ要約**: 新着トリガ + デバウンスで「今の状況」ロールアップを自動更新、列ヘッダに表示。
- **深堀りチャット**: 35B が vault_search(ハイブリッド検索 + 時間減衰)と web_search を自律的に使い、出典 URL 付きで回答。記事からもトップバーからも起動可能。
- **運用**: start.bat / stop.bat。Electron 単体起動でもバックエンドと llama-server を自動起動。ウィンドウを閉じてもトレイ常駐で取り込み継続。SQLite 索引は MD から全再構築可能(POST /admin/rebuild-index)。
