# 検索クエリ設計ガイド

作成日時: 2026-07-05 02:08
更新日時: 2026-07-05 02:08

カテゴリの `query_templates` を設計・調整するためのガイド。2026-07-05 の
ローカルLLM カテゴリの調査(ヒット激減の原因究明)で得た実測に基づく。

---

## 1. 検索の仕組み(前提知識)

- 取り込みは **ddgs のニュース検索**(`server/search_web.py` の `search_news`)。
  DuckDuckGo news → Bing news → Yahoo news のバックエンドを ddgs が自動で使い分ける。
- 既定パラメータ: `region="jp-jp"`、`timelimit="w"`(過去1週間)、最大20件。
- カテゴリごとに `poll_interval_sec`(+ジッタ)の間隔で `query_templates` を
  **1本ずつローテーション**して検索する。テンプレート中の `{month}` は
  「2026年7月」形式に展開される。
- 重複は URL 正規化 + tombstone で dedup されるため、**クエリ同士の結果が
  重なっても害はない**。広めに取ってよい。

## 2. 設計原則

### 原則1: ニッチな専門用語はニュース索引に存在しない

ニュース検索は「ニュースメディアが報じた記事」の索引である。技術コミュニティ
(Zenn / Qiita / Reddit / HF 等)でしか使われない語はヒットしない。

実測(2026-07-05、timelimit=w):

| クエリ | ヒット数 | 備考 |
|---|---|---|
| `新しい GGUF モデル` | 0 | 専門用語 |
| `llama.cpp release` | 0 | 専門用語 |
| `量子化 手法` | 2 | 全て量子コンピュータの誤ヒット |
| `Qwen 新モデル リリース` | 0 | 語の組み合わせが限定的すぎ |
| `local LLM` | **15** | 一般語・英語 |
| `Hugging Face` | **11** | 固有名詞 (ノイズ含む) |
| `open source LLM model release` | **9** | 一般語・英語 |
| `ローカルLLM` | 6 | 一般語・日本語 |
| `Ollama` | 5 | 知名度の高い固有名詞 |

### 原則2: 技術系トピックは英語クエリが圧倒的に強い

海外テック媒体の記事量は日本語媒体の数倍〜数十倍。英語見出しは
環境設定「見出しの日本語訳」が吸収するので、英語クエリを恐れない。

### 原則3: ノイズはクエリで防がず、キュレーションに任せる

`Hugging Face` はイラン情勢、`Gemma 新モデル` はファッション、`Claude` は
ホッケー選手を拾うことがある。これをクエリの絞り込みで防ごうとすると
ヒット数が激減する(原則1の罠に落ちる)。**クエリは広く、除去は 9B の
関連度採点(ノイズ閾値)で行う**のがこのアプリの設計。

### 原則4: 1カテゴリ 4〜7 本、性格の違うクエリを混ぜる

- 固有名詞系(`Ollama`, `NVDA 決算`): 精度が高い
- 一般語系(`local LLM`, `GPU 供給 価格`): 網羅性が高い
- 時事系(`HBM 需給 {month}`): 鮮度が高い

ローテーションは1周 `poll_interval_sec × 本数` かかる点に注意
(例: 600秒 × 7本 = 70分)。本数を増やすほど1本あたりの巡回が遅くなる。

## 3. クエリを診断・調整する手順

1. 症状: カテゴリの新着が極端に少ない/無関係な記事ばかり。
2. 計測: 各クエリを `search_news` に直接かけてヒット数とタイトルを見る。

```python
from server.search_web import search_news
for q in ["候補クエリ1", "候補クエリ2"]:
    hits = search_news(q, max_results=15)
    print(len(hits), q, [h["title"][:40] for h in hits[:3]])
```

3. 0件が続くクエリは「より一般的な語」「英語」に言い換えて再計測。
4. 勝ったクエリをカテゴリ設定(歯車アイコン)に反映。
5. 反映後は `POST /admin/ingest-now?category=<id>` を数回叩いて
   ローテーションを進め、実際の取り込みを確認する。

※ 連続で叩くと DuckDuckGo news が一時的に 403 を返すことがある
(ddgs が Bing/Yahoo に自動フォールバックするが、計測時は2秒程度の
間隔を空けると安定する)。

## 4. 各カテゴリの現行クエリの狙い(2026-07-05 時点)

- **semiconductor-stocks**: `HBM 需給 {month}` / `TSMC 稼働率` / `NVDA 決算` /
  `対中 半導体 規制` — 投資判断系は日本語媒体が厚く、日本語で成立している。
- **local-llm**: `local LLM` / `open source LLM model release` / `ローカルLLM` /
  `Ollama` / `Hugging Face` / `Qwen model` / `Gemma model` — 原則1〜3 を
  全面適用した英語中心セット(旧セットは0件が多発したため置き換え)。
- **gpu / ai**: 日本語中心だが現状ヒット良好。少なくなったら同じ手順で診断。
